## Дизайн-система (существующая)

Используй токены из `design_handoff_sweethome 4/README.md` и `data.js` / `dark.js`. Все новые экраны должны быть в рамках существующей системы.

### Ключевые токены

**Цвета:**
- Primary: `#5B7C5A` (sage green, light) / `#9CC09A` (dark)
- Accent: `#E8A87C` (terracotta)
- Background: `#FBF7F4` (light) / `#0F1310` (dark)
- Surface: `#FFFFFF` (light) / `#1A1F1B` (dark)
- Error/Destructive: `#D4574E` (light) / `#E57368` (dark)

**Типографика:** Inter, 11 уровней (micro → display)  
**Отступы:** 4 / 6 / 8 / 10 / 12 / 14 / 16 / 20 / 24 dp  
**Радиусы:** кнопки 16, карточки 16, пилюли 20–22

### G-01: Переработка Dashboard (Главный экран)

**Приоритет:** 🔴 MVP

#### Структура экрана (сверху вниз)

```
TopAppBar
├── [Логотип/название]
└── [Иконка уведомлений] [Иконка профиля]

ContextPill (центр или слева)
├── Состояние 1: [🏢 Работа]  ← активно в рабочие часы
└── Состояние 2: [🏠 Личное]  ← активно вечером/выходные
    Переключается тапом. Анимация: slide + fade

Section "Сегодня"
├── Заголовок: "Сегодня" + счётчик задач
├── Просроченные (если есть): красная метка "Просрочено N"
├── TaskCard (compact) × N
│   ├── Чекбокс
│   ├── Название задачи
│   ├── Метки: список (chip), приоритет (dot), dueAt (time)
│   └── AssigneeChip (в групповых)
└── Пустое состояние: иллюстрация + "На сегодня всё выполнено 🎉"

Section "Цели" (placeholder в MVP)
└── Card с надписью "Цели — скоро" (без замка, нейтральный стиль)

FAB (нижний правый)
└── Раскрывается в 3 действия:
    ├── + Задача
    ├── 🛒 Список покупок
    └── 🏠 Домашнее дело
```

#### Лимит «Сегодня»

Показываем первые **3** задачи (сортировка: просроченные → high → medium → low).  
Если задач больше: кнопка-строка **«Ещё N задач →»** под списком → разворачивает все.

#### Состояния

- **Work workspace не создан:** контекст-пилюля скрыта, показываются personal-задачи
- **Рабочие часы, есть задачи:** контекст-пилюля «Работа», список рабочих задач
- **Личное время, есть задачи:** контекст-пилюля «Личное», список личных задач
- **Нет задач на сегодня:** empty state с иллюстрацией
- **Есть просроченные:** бейдж с числом в заголовке секции

#### Компоненты

**ContextPill:**
- Размер: высота 32dp, padding h 16dp
- Иконка (16dp) + текст (Body/14)
- Active background: Primary Container
- Inactive: Surface с border
- Анимация переключения: 200ms

**TaskCard (compact):**
- Высота: 56–64dp
- Чекбокс → анимация выполнения (strike-through + fade)
- Цветная точка приоритета: high=красный, medium=янтарный, low=серый

### G-02: Экран создания задачи (динамические поля)

**Приоритет:** 🔴 MVP

#### UX-паттерн

Bottom sheet на 85% высоты экрана (swipeable). Поднимается при тапе на FAB или на поле добавления в ListDetail.

#### Структура

```
Handle bar (drag indicator)
Header
├── [✕ Закрыть]
├── "Новая задача" / "Редактировать задачу"
└── [Сохранить] (primary button, disabled until title filled)

─── БАЗОВЫЕ ПОЛЯ (всегда показаны) ───
[TextField] Название задачи *required*
[TextField] Заметка (optional, multiline)
[Row] Приоритет: [Высокий] [Средний] [Низкий] (chip group)
[DatePicker] Дедлайн: "Добавить дату →"
[AssigneePicker] Исполнитель: "Назначить →" (только в групповых)

─── ШАБЛОНЫ ───
[Button outline] 📋 Использовать шаблон
    → открывает G-03 поверх

─── ДИНАМИЧЕСКИЕ СЕКЦИИ ───

[РАСПИСАНИЕ] — показывается только если listType = home_chores
Radio group (один режим за раз):
◉ Одноразово         → без дополнительных полей
○ Каждые N дней      → показать: [NumberField "N"] + дата начала
○ По дням недели     → показать: [Пн][Вт][Ср][Чт][Пт][Сб][Вс] + дата начала

[ДЕТАЛИ ТОВАРА] — показывается только если listType = shopping
ExpandableSection "Детали товара"
├── [TextField] Бренд / марка
├── [ImagePicker] Фото → "Добавить фото"
└── [TextField] Ссылка на товар

[НАГРАДА] — показывается только если workspace = family / mentoring
Row "Награда"
├── [NumberField] Количество монет
└── Валюта: "🪙 Монеты" (название из HomeCurrency)

[MEDIA] — показывается только если listType = media
Row "Тип медиа": [Книга][Фильм][Сериал][Игра][Статья][Подкаст]
Row "Статус": [Хочу][В процессе][Выполнено]
[TextField] URL / ссылка
[RatingBar] Оценка 1–5 ⭐
```

#### Состояния полей

- При выборе типа списка: динамические секции появляются с анимацией (slide down + fade, 200ms)
- Обязательные поля: border становится Primary при фокусе, Error при попытке сохранить пустым
- **Фото товара (imageUrl) — не в MVP.** В MVP только текстовые поля «Бренд» и «Ссылка»

### G-03: Выбор шаблона (inline bottom sheet)

**Приоритет:** 🔴 MVP

#### Структура

Bottom sheet поверх G-02, 70% высоты:

```
Header: "Выбрать шаблон"  [✕]
SearchField: "Поиск шаблонов..."

TabRow: [Шаблоны] [Часто] [Избранное]

─── Вкладка "Шаблоны" ───
TemplateCard × N
├── Иконка (32dp)
├── Название
├── Краткое описание (TextSecondary, 12sp)
└── Тип: chip (home_chores / shopping / etc.)

─── Вкладка "Часто добавляете" ───
ItemChip × N (компактные, горизонтальный scroll)

─── Вкладка "Избранное" ───
ItemRow × N
```

**Фильтрация шаблонов:** показывать только совместимые с текущим типом списка. Если список `home_chores` — показывать только chore-шаблоны. Для `shopping` — частые покупки и избранное.

**При тапе на шаблон:**
- Bottom sheet закрывается
- Форма G-02 заполняется данными шаблона
- Все поля доступны для редактирования

### G-04: Детали элемента в Shopping List

**Приоритет:** 🔴 MVP

#### Реализация: chevron + bottom sheet при тапе

```
ItemRow (обычный вид — нет деталей)
├── Checkbox
├── Название
└── Количество + единица

ItemRow (есть brand или productUrl)
├── Checkbox
├── Название
├── Количество + единица
└── [›] Chevron иконка справа (16dp, TextSecondary цвет)

Тап на строку с chevron → Bottom Sheet "Детали товара":
├── Название (Headline/16)
├── Бренд: "Nike" (если есть)
├── Количество + единица
├── [🔗 Ссылка на товар →] (если есть, кликабельная)
├── Divider
└── [✏️ Редактировать] [🗑️ Удалить] (actions)
```
> **MVP**: фото не отображается (imageUrl — MMP)

#### Отклонён вариант: Expandable row

```
ItemRow collapsed:
└── [⌄] стрелка справа если есть детали

ItemRow expanded:
├── Основная строка (как обычно)
└── Expanded area:
    ├── Thumbnail (если есть)
    ├── Бренд, ссылка
    └── [Редактировать детали]
```

**Рекомендация: Вариант A (bottom sheet)** — список остаётся компактным.

### G-06: AssigneeChip — Quick-assign

**Приоритет:** 🔴 MVP

#### Состояния компонента (48×24dp / compact)

```
unassigned:
└── [👤+] Серый фон, иконка "person_add"

assigned_me:
└── [АБ] Primary bg, инициалы текущего пользователя

assigned_other:
└── [АБ] SurfaceVariant bg, инициалы другого участника
```

#### Поведение

| Состояние | Тап |
|-----------|-----|
| `unassigned` | Назначить себя → анимация смены состояния + Snackbar «Назначено вам · Отменить» (3 сек) |
| `assigned_me` | Bottom sheet: снять с себя / назначить другого |
| `assigned_other` | Bottom sheet: назначить себя / изменить / снять |

**Bottom sheet выбора исполнителя:**
```
"Назначить исполнителя"  [✕]
[Моё имя + аватар] (наверху, с бейджем "Я")
[Участник 2]
[Участник 3]
[Без исполнителя]
```

### G-07: Удаление аккаунта (Danger Zone)

**Приоритет:** 🔴 MVP

#### В настройках профиля

```
Settings Screen:
...другие секции...
─────────────────────
Секция "Danger Zone"  ← красный заголовок
[🗑️ Удалить аккаунт]  ← outlined кнопка с Error цветом
                       ← disabled + tooltip "Нет подключения к сети" если офлайн
```

#### Confirmation flow (2 шага)

**Шаг 1 — Предупреждение (Modal / Bottom Sheet):**
```
Иконка ⚠️ (большая, Error цвет)
"Удалить аккаунт?"
"Это действие нельзя отменить. Все ваши данные, списки и история будут удалены безвозвратно."
[Отмена]  [Продолжить →]  ← Error цвет
```

**Шаг 2 — Подтверждение:**
```
"Введите email для подтверждения"
[TextField] ваш@email.com
[Отмена]  [Удалить навсегда]  ← заблокировано, пока email не совпадает
```

**Success state:**
```
Иллюстрация (прощальная)
"Аккаунт удалён"
"Все данные удалены. Спасибо, что были с нами."
[Закрыть]  ← возврат на экран авторизации
```

### G-10: Список из меню / рецепта

**Приоритет:** 🟡 MMP

#### Entry point

В экране «Списки» или при создании нового списка типа `shopping`: кнопка «Создать из рецепта».

#### Флоу

```
Step 1 — Ввод рецепта:
"Вставьте рецепт или текст меню"
[TextArea] — большое поле для текста
[Выбрать шаблон рецепта →]  ← шаблоны: Борщ, Паста, Пицца...
[Далее →]

Step 2 — Результат парсинга:
"Найдено ингредиентов: 8"
[ItemRow × N]  ← каждый ингредиент с чекбоксом
├── Мука — 500г
├── Яйца — 3 шт
...
[Убрать ненужные]
[Создать список покупок]

Step 3 — Название списка:
[TextField] "Список из рецепта: Паста карбонара"
[Сохранить]
```
